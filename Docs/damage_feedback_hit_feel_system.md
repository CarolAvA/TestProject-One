# 打击表现系统方案

## 模块名：Damage Feedback / Hit Feel / 受击表现系统

---

## 1. 系统定位

这一套系统专门负责 **AI角色造成伤害**、**怪物受到伤害**、**AI角色被造物主反制** 时的表现反馈。

目标不是简单显示数字，而是让玩家一眼看懂：

1. 这次是什么元素伤害。
2. 是普通伤害还是暴击。
3. 怪物有没有被打出停顿、击退、硬直。
4. 当前AI角色攻击节奏是不是很强。
5. AI角色被造物主压制时会不会有情绪反应。
6. 整个战斗看起来是不是有节奏、有声音、有打击感。

最终表现流程：

```text
AI弹幕命中怪物
↓
命中音效响起
↓
怪物闪白 / 闪红 / 元素闪光
↓
怪物轻微停顿 + 后退
↓
伤害跳字突然放大出现
↓
跳字跟随节奏上飘、缩小、变淡
↓
AI角色或怪物触发吐槽冒泡
```

---

## 2. 打击表现系统总结构

```text
HitFeedbackSystem
├── DamageNumberSystem        跳字系统
├── HitFlashSystem            受击闪光系统
├── HitStopSystem             受击停顿系统
├── KnockbackSystem           击退系统
├── HitVFXSystem              命中特效系统
├── HitSFXSystem              命中音效系统
├── BubbleTalkSystem          角色冒泡吐槽系统
├── RhythmHitSyncSystem       节奏同步系统
└── FeedbackConfigDatabase    表现配置库
```

# 3. 伤害跳字系统

## 3.1 跳字系统定位

伤害跳字不只是显示数值，而是战斗反馈的重要组成部分。

跳字需要表达：

| 信息 | 表现方式 |
|---|---|
| 伤害类型 | 不同元素颜色 |
| 普通伤害 | 标准字号、普通描边 |
| 暴击伤害 | 更大字号、更粗描边、更强缩放 |
| DOT持续伤害 | 小字号、连续跳出 |
| AOE伤害 | 多个单位同时跳字，略有随机偏移 |
| 破盾伤害 | 特殊破盾字体 / 碎裂效果 |
| 治疗 | 绿色向上跳字 |
| 免疫 | 显示“免疫” |
| 闪避 | 显示“闪避” |
| 格挡 | 显示“格挡” |
| 弱点伤害 | 显示“弱点”或特殊颜色 |
| 克制伤害 | 显示“克制”标签 |

---

## 3.2 元素伤害颜色规则

| 伤害类型 | 跳字颜色 | 推荐视觉 |
|---|---|---|
| 物理伤害 | 白色 / 浅灰 | 干净、直接 |
| 火焰伤害 | 橙红色 | 字体边缘带火星 |
| 冰霜伤害 | 浅蓝色 | 字体边缘带冰晶 |
| 雷电伤害 | 蓝白 / 紫蓝 | 字体闪电抖动 |
| 毒气伤害 | 绿色 / 黄绿色 | 字体带毒泡 |
| 音波伤害 | 紫色 / 粉紫 | 字体带声波环 |
| 暗影伤害 | 深紫 / 黑紫 | 字体带暗雾 |
| 神圣伤害 | 金白色 | 字体带光晕 |
| 真实伤害 | 红白色 | 不受抗性影响，醒目 |
| 破盾伤害 | 青蓝碎裂色 | 带碎盾效果 |
| 治疗 | 绿色 | 向上飘，柔和 |
| 吸血回血 | 暗红 + 绿色 | 先伤害后回血 |
| 免疫 | 灰色 | 不显示数值，只显示“免疫” |
| 闪避 | 浅灰 | 文字快速闪过 |
| 格挡 | 金属灰 | 带盾牌图标 |

---

## 3.3 普通伤害和暴击伤害差异

### 普通伤害

| 项目 | 规则 |
|---|---|
| 字号 | 标准 |
| 描边 | 细描边 |
| 出现动画 | 快速放大到110% |
| 持续时间 | 0.8 ~ 1.0秒 |
| 飘动方向 | 向上轻微随机 |
| 透明度 | 后半段渐隐 |
| 音效 | 普通命中音 |

表现节奏：

```text
出现：0% → 110% → 100%
停留：短暂
结束：向上飘 + 缩小 + 淡出
```

---

### 暴击伤害

| 项目 | 规则 |
|---|---|
| 字号 | 普通伤害的1.35 ~ 1.6倍 |
| 描边 | 粗描边 |
| 字体 | 更厚重，可以轻微倾斜 |
| 出现动画 | 瞬间爆开放大到150% |
| 持续时间 | 1.2 ~ 1.6秒 |
| 飘动方向 | 向上并轻微左右摆动 |
| 透明度 | 更晚开始淡出 |
| 音效 | 更重的命中音 / 暴击音 |
| 特效 | 字体背后有冲击闪光 |

表现节奏：

```text
出现：0% → 160% → 120% → 100%
停留：略久
结束：向上飘 + 慢慢变小 + 淡出
```

---

## 3.4 跳字动画节奏

跳字要有“节奏感”，不要只是线性上飘。

推荐分4段：

```text
第1段：爆点出现
第2段：回弹稳定
第3段：上飘缩小
第4段：淡出消失
```

### 普通跳字时间轴

| 时间 | 表现 |
|---|---|
| 0.00s | 数字出现，Scale = 0 |
| 0.08s | 突然放大到1.15 |
| 0.16s | 回弹到1.0 |
| 0.50s | 开始向上漂移 |
| 0.75s | Scale变为0.85 |
| 0.95s | Alpha变为0，销毁 |

### 暴击跳字时间轴

| 时间 | 表现 |
|---|---|
| 0.00s | 数字出现，Scale = 0 |
| 0.06s | 爆炸放大到1.6 |
| 0.15s | 回弹到1.25 |
| 0.30s | 稳定到1.1 |
| 0.80s | 向上漂移，轻微左右摆动 |
| 1.20s | Scale变为0.9 |
| 1.50s | Alpha变为0，销毁 |

---

## 3.5 跳字运动规则

单个跳字运动：

```text
初始位置：目标单位头顶附近
↓
随机左右偏移：-20px ~ +20px
↓
突然放大
↓
向上飘动
↓
轻微横向漂移
↓
缩小
↓
淡出
```

多个跳字同时出现时：

```text
第一个：向上
第二个：向左上
第三个：向右上
第四个：更高位置
第五个以后：合并或延迟显示
```

避免所有数字重叠在一起。

---

## 3.6 跳字合并规则

如果短时间内同一个目标受到多次小额伤害，需要防止满屏数字。

| 情况 | 处理 |
|---|---|
| 0.2秒内同类型DOT伤害 | 合并为一个跳字 |
| 0.3秒内多个小伤害 | 合并显示总伤害 |
| 暴击伤害 | 不合并，单独显示 |
| 破盾伤害 | 不合并，单独显示 |
| 免疫 / 闪避 | 不合并，单独显示 |
| 元素不同 | 不合并，分别显示 |
| 同一次AOE命中多个怪 | 每个怪独立显示，但位置错开 |

---

## 3.7 DOT持续伤害跳字

毒、燃烧、流血等持续伤害不能太抢眼。

| 项目 | 规则 |
|---|---|
| 字号 | 普通伤害的70% |
| 动画 | 轻微上飘 |
| 持续时间 | 0.6秒 |
| 颜色 | 根据元素 |
| 描边 | 更弱 |
| 频率 | 跟TickInterval同步 |
| 合并 | 同一目标短时间可合并 |

示例：

```text
毒伤：绿色小数字，每秒跳一次
燃烧：橙色小数字，每0.5秒跳一次
流血：暗红小数字，每0.8秒跳一次
```

---

## 3.8 特殊跳字类型

| 类型 | 文本 | 表现 |
|---|---|---|
| 免疫 | 免疫 | 灰色，快速弹出后消失 |
| 闪避 | 闪避 | 浅灰，向侧面滑走 |
| 格挡 | 格挡 | 金属色，带盾牌小图标 |
| 破盾 | 破盾 | 青蓝碎裂字效 |
| 弱点 | 弱点！ | 黄色 / 红色，带爆点 |
| 克制 | 克制！ | 元素颜色 + 小标签 |
| 吸收 | 吸收 | 蓝色，向角色体内收缩 |
| 治疗 | +120 | 绿色，向上柔和漂浮 |
| 吸血 | +80 | 暗红转绿色 |
| 击杀 | 击杀 | 大字短暂出现 |
| 精英受创 | 重创！ | 更大、更重 |

# 5. 受击闪光系统

## 5.1 系统定位

怪物受击时，序列帧图片需要产生明显反馈。

最基础表现：

1. 闪白。
2. 闪红。
3. 根据元素闪对应颜色。
4. 短暂停顿。
5. 后退。
6. 播放命中特效。
7. 播放命中音效。

---

## 5.2 闪光颜色规则

| 伤害类型 | 闪光颜色 | 表现 |
|---|---|---|
| 物理 | 白色 | 快速闪白 |
| 火焰 | 橙红色 | 闪红 + 火星 |
| 冰霜 | 浅蓝色 | 闪蓝 + 冰晶 |
| 雷电 | 蓝白色 | 闪白 + 电弧 |
| 毒气 | 绿色 | 闪绿 + 毒雾 |
| 音波 | 紫色 | 闪紫 + 波纹 |
| 暗影 | 深紫 | 暗化 + 紫闪 |
| 神圣 | 金白色 | 金光闪烁 |
| 暴击 | 白色加强 + 红描边 | 更强闪烁 |
| 破盾 | 青蓝裂纹 | 护盾碎裂闪 |

---

## 5.3 闪光持续时间

| 类型 | 时间 |
|---|---:|
| 普通受击 | 0.08 ~ 0.12秒 |
| 元素受击 | 0.12 ~ 0.18秒 |
| 暴击受击 | 0.18 ~ 0.25秒 |
| 精英受击 | 0.1 ~ 0.16秒 |
| Boss受击 | 0.06 ~ 0.1秒，避免太频繁刺眼 |
| 死亡一击 | 0.2秒 + 死亡特效 |

---

## 5.4 闪光方式

### 方式一：材质闪白

适合Unity 2D SpriteRenderer。

```text
受击
↓
材质颜色切换为白色 / 元素色
↓
保持0.08秒
↓
恢复原色
```

### 方式二：叠加一张白色遮罩

适合序列帧角色。

```text
原序列帧
↓
上层叠加同形状白色Sprite
↓
透明度从1快速降到0
```

### 方式三：Shader控制

字段：

```text
HitFlashColor
HitFlashIntensity
HitFlashDuration
HitFlashBlendMode
```

推荐使用Shader，因为可以批量控制，也方便表现不同元素。

# 6. 受击停顿 Hit Stop

## 6.1 系统定位

受击停顿是打击感核心。

不是所有伤害都停顿，只有明显命中、暴击、重击、控制、Boss技能需要。

---

## 6.2 停顿类型

| 类型 | 说明 |
|---|---|
| 受击者停顿 | 被打的怪物停一下 |
| 攻击者停顿 | AI角色攻击命中时轻微顿一下 |
| 全局短暂停顿 | 暴击、重击、大招时整个画面极短暂停 |
| 局部时间缩放 | 只影响附近单位，不影响全场 |

---

## 6.3 停顿时间规则

| 命中类型 | 停顿时间 |
|---|---:|
| 普通小伤害 | 0 ~ 0.03秒 |
| 普通子弹命中 | 0.03 ~ 0.05秒 |
| 元素弹命中 | 0.04 ~ 0.07秒 |
| 暴击 | 0.08 ~ 0.12秒 |
| 重击 | 0.1 ~ 0.16秒 |
| 击杀 | 0.06 ~ 0.1秒 |
| Boss重击 | 0.12 ~ 0.2秒 |
| 高频DOT | 不触发停顿 |
| 高频弹幕 | 每0.2秒最多触发一次停顿 |

---

## 6.4 停顿频率限制

必须限制，否则高频弹幕会把画面打卡。

```text
同一目标HitStop最小间隔：0.15秒
全局HitStop最小间隔：0.4秒
高频弹幕HitStop强度降低50%
DOT不触发HitStop
```

|---|---|

## 9.2 高频音效限制

高频弹幕不能每一发都完整播放命中音，否则会很吵。

规则：

```text
同类命中音效0.08秒内最多播放1次
同一目标命中音效0.12秒内最多播放1次
高频弹幕命中音量降低20%
DOT伤害只播放Tick轻音，或者每3次播放1次
暴击音效不被普通命中音覆盖
```

# 10. AI角色受击冒泡吐槽系统

## 10.1 系统定位

当AI角色受到造物主技能、怪物攻击、控制、陷阱影响时，在角色头顶出现随机吐槽冒泡。

效果目标：

> 让AI角色像一个真人一样，会抱怨、咒骂、挑衅造物主，让玩家感觉自己真的在和一个“会反应的英雄”对抗。

---

## 10.2 冒泡显示规则

| 项目 | 规则 |
|---|---|
| 显示位置 | AI角色头顶附近 |
| 随机偏移 | 左右 -60px ~ +60px，上下 20px ~ 80px |
| 同时数量 | 最多2个 |
| 持续时间 | 1.5 ~ 2.5秒 |
| 出现动画 | 弹出放大 |
| 消失动画 | 缩小 + 淡出 |
| 触发间隔 | 普通触发至少间隔4秒 |
| 高优先级事件 | 可打断普通吐槽 |
| 文字长度 | 4~12字为主 |
| 气泡方向 | 尖角指向AI角色 |
| 多次触发 | 从内容库随机，不要重复太频繁 |

---

## 10.3 冒泡位置随机

```text
AI角色头顶中心
↓
随机X偏移：-60 ~ +60
↓
随机Y偏移：40 ~ 100
↓
如果超出屏幕边缘，自动往屏幕内偏移
↓
如果已有冒泡，新的冒泡向另一侧偏移
```

---

## 10.4 冒泡触发条件

| 触发事件 | 说明 |
|---|---|
| 被造物主技能命中 | 最常见吐槽 |
| 被高伤害技能命中 | 强烈吐槽 |

# 14. 打击表现与音乐节奏结合

## 14.1 节拍点命中反馈

AI的伤害来自音效节拍器，所以打击反馈也应该跟节奏绑定。

```text
节拍点触发
↓
发射弹幕
↓
播放节拍音效
↓
命中怪物
↓
跳字在节拍点后0.02~0.05秒出现
↓
命中特效和音效同步
↓
怪物闪光和停顿同步
```

---

## 14.2 不同音轨对应不同打击表现

| 音轨 | 打击表现 |
|---|---|
| 高音轨 | 小跳字、快速、轻闪、连续感 |
| 中音轨 | 标准跳字、标准命中特效 |
| 低音轨 | 大跳字、强缩放、重击停顿、震屏 |
| 重音点 | 暴击感、描边、强特效 |
| 断拍 | 突然停顿，适合控制技能 |
| 连拍 | 多个小跳字快速出现 |
| 终曲 | 大范围跳字 + 强镜头反馈 |

# 16. 打击反馈数据流

```text
AI节奏器触发BeatEvent
↓
发射弹道 / 释放AOE
↓
命中怪物
↓
DamageSystem生成DamageResult
↓
HitFeedbackSystem读取DamageResult
↓
并行触发：
    1. DamageNumberSystem 显示跳字
    2. HitFlashSystem 闪白 / 元素闪
    3. HitStopSystem 停顿
    4. KnockbackSystem 击退
    5. HitVFXSystem 命中特效
    6. HitSFXSystem 命中音效
↓
如果AI角色被攻击：
    BubbleTalkSystem 判断是否触发吐槽
```

# 21. 最终效果目标

这套打击表现系统要达到的体验是：

```text
看跳字颜色 → 知道伤害元素
看跳字大小 → 知道是否暴击
看跳字动画 → 感觉攻击有节奏
看怪物闪光 → 知道命中有效
看停顿后退 → 感觉攻击有力量
看命中特效 → 知道技能类型
听命中音效 → 感受节拍反馈
看AI冒泡 → 感觉AI像真人在对抗玩家
```

最终玩家应该觉得：

> AI不是一个冷冰冰的自动角色，而是一个会攻击、会成长、会躲避、会吐槽、会被造物主折磨到急眼的“活人英雄”。
