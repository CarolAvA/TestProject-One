# 《击败音乐狂》完整UI布局说明文档
## 反幸存者玩法｜造物主操控｜AI音乐狂人｜2D像素搞怪音乐风

---

# 0. 文档目标

本文档用于定义《击败音乐狂》的完整UI布局规则。

核心目标：

1. **主战斗区域不能被UI遮挡。**
2. **玩家操作按钮必须清晰、好点、好理解。**
3. **AI当前状态、节奏、BD、技能预警必须清楚。**
4. **造物主方的怪物召唤、技能释放、能量消耗必须直观。**
5. **整体界面要美观，符合搞怪音乐像素风。**
6. **移动端和PC端都能适配。**
7. **信息层级清晰，不堆满屏幕。**

一句话原则：

> 中央留给战斗，底部负责操作，顶部负责状态，左右只放轻量信息和可折叠面板。

---

# 1. UI整体设计原则

## 1.1 主要视觉结构

游戏战斗界面采用 **“中间战斗区 + 底部操作区 + 顶部状态区 + 左右辅助区”** 的结构。

```text
┌──────────────────────────────────────────────┐
│ 顶部状态区：AI血量 / 时间 / 资源 / 威胁值       │
├───────────────┬────────────────┬─────────────┤
│ 左侧轻量信息区 │    主战斗区域     │ 右侧轻量信息区 │
│ 任务/提示/小图 │   绝不遮挡核心战斗 │ AI状态/预警    │
├───────────────┴────────────────┴─────────────┤
│ 底部节奏信息区：AI音效进度条 + 当前BD列表       │
├──────────────────────────────────────────────┤
│ 底部操作区：怪物抽屉 + 造物主技能 + 能量条       │
└──────────────────────────────────────────────┘
```

---

## 1.2 主战斗区域保护规则

主战斗区域是玩家观察AI移动、怪物包围、弹幕、技能预警的核心区域。  
所有UI都必须给战斗留空间。

### 必须保留的战斗可视区域

| 屏幕比例 | 主战斗区建议占比 |
|---|---:|
| 16:9 PC | 70% ~ 76% |
| 16:9 手机横屏 | 66% ~ 72% |
| 19.5:9 手机横屏 | 68% ~ 74% |
| 平板横屏 | 72% ~ 78% |

### 禁止遮挡区域

```text
屏幕中央 70%宽 × 60%高 区域
禁止放置：
- 大型弹窗
- 固定技能栏
- 固定信息面板
- 常驻Tips
- 大型头像
- 任务长文本
```

---

## 1.3 信息优先级

战斗中信息非常多，必须分层。

| 优先级 | 信息 | 显示方式 |
|---|---|---|
| S级 | AI血量、倒计时、玩家能量、技能冷却、危险预警 | 常驻显示 |
| A级 | AI当前攻击节奏、当前BD、造物主技能范围 | 常驻或半常驻 |
| B级 | 怪物召唤列表、当前克制提示、Boss状态 | 折叠 / 弹出 |
| C级 | Tips详情、BD详细说明、怪物详情 | 悬停 / 长按显示 |
| D级 | 教程提示、任务说明 | 可关闭 / 自动收起 |

---

## 1.4 UI风格原则

UI风格符合“搞怪像素音乐演唱会”。

关键词：

- 像素风
- 黑灰底
- 霓虹边
- 音乐播放器感
- DJ控制台感
- 演出海报贴纸感
- 搞怪但清晰
- 不做厚重写实UI
- 不做古风UI

# 3. 顶部状态区 TopStatusBar

## 3.1 顶部状态区作用

顶部只显示战斗最核心状态：

1. AI角色头像
2. AI血条
3. AI等级
4. AI当前威胁值
5. 战斗时间
6. 造物主能量
7. 暂停按钮
8. 可选：当前关卡目标

---

## 3.2 布局结构

```text
┌──────────────────────────────────────────────┐
│ [AI头像] [AI血条 Lv.8]   [05:32]   [威胁70%] [能量 120/200] [暂停] │
└──────────────────────────────────────────────┘
```

---

## 3.3 顶部状态区字段

| UI元素 | 位置 | 说明 |
|---|---|---|
| AI头像 | 左上 | 显示音乐狂人头像，低血时头像变表情 |
| AI血条 | 头像右侧 | 主血条，必须醒目 |
| AI等级 | 血条下方或右侧 | 显示当前等级 |
| 当前主流派 | 血条下方小标签 | 例如：雷电连奏、火焰爆裂 |
| 战斗时间 | 顶部中间 | 显示本局时间 |
| 威胁值 | 中右 | 显示AI成长压力 |
| 造物主能量 | 右侧 | 玩家召唤和技能资源 |
| 暂停按钮 | 最右 | 固定按钮，不遮挡 |

---

## 3.4 AI血条规则

### 正常状态

```text
[头像] ████████████  85%
        Lv.8  雷电连奏
```

### 低血状态

- 血条闪红
- AI头像切换为紧张表情
- 可显示“濒危”标签
- 音乐狂人冒泡吐槽优先级提高

### 护盾状态

如果AI有护盾：

```text
护盾条在血条上层或下层显示
蓝色细条，破盾时出现裂纹动画
```

---

## 3.5 威胁值显示

威胁值不是普通数值，应该用“舞台热度计”表现。

```text
威胁值：▮▮▮▮▮▯  70%
```

视觉风格：

- 低威胁：蓝紫色
- 中威胁：黄色
- 高威胁：橙色
- 极高威胁：红色闪烁

# 4. 主战斗区域 MainBattleView

## 4.1 主战斗区原则

主战斗区域只放战斗对象和必要的战斗提示。

允许显示：

1. AI角色
2. 怪物
3. 弹幕
4. 地面预警
5. 技能范围
6. 受击跳字
7. Buff图标
8. 冒泡吐槽
9. 召唤落点提示

不允许长期显示：

1. 大型信息框
2. 大型角色详情
3. 大型BD Tips
4. 大面积半透明面板
5. 长文本任务说明

---

## 4.2 主战斗区UI叠层顺序

从下到上：

```text
地形Tile
↓
地面装饰
↓
陷阱 / 区域持续效果
↓
造物主技能预警
↓
怪物 / AI角色
↓
弹道 / 弹幕
↓
命中特效
↓
跳字
↓
Buff小图标
↓
AI冒泡吐槽
↓
临时战斗提示
```

---

## 4.3 召唤落点提示

玩家选择怪物后，点击地图召唤。

### 可召唤区域

- 地面出现绿色/紫色召唤圈
- 鼠标/手指拖动时跟随显示
- 允许召唤时图标明亮

### 不可召唤区域

- 显示红色叉
- 提示原因：
  - 距离AI太近
  - 能量不足
  - 区域被阻挡
  - 数量达到上限

---

## 4.4 造物主技能预警

造物主技能释放时必须清楚。

| 技能形状 | UI表现 |
|---|---|
| 圆形 | 半透明圆圈 + 边缘脉冲 |
| 扇形 | 扇形区域 + 方向箭头 |
| 直线 | 细长路径 + 起点终点 |
| 多点 | 多个落点圈 |
| 跟随目标 | 预警圈跟随AI，但释放前锁定 |
| 全屏 | 屏幕边缘危险闪烁，不遮挡中央 |

---

# 5. 左侧轻量信息区 LeftInfoPanel

## 5.1 左侧信息用途

左侧只放辅助信息，不放核心操作按钮。

显示内容：

1. 当前关卡目标
2. 简短教程提示
3. 当前推荐反制提示
4. 危险事件提示
5. 最近系统消息

---

## 5.2 布局结构

```text
┌──────────────┐
│ 当前目标       │
│ 击败音乐狂人   │
├──────────────┤
│ 提示：         │
│ AI正在高频弹幕 │
│ 小怪容易被清理 │
└──────────────┘
```

---

## 5.3 显示规则

| 内容 | 显示方式 |
|---|---|
| 关卡目标 | 常驻小面板 |
| 新手提示 | 自动收起 |
| 反制建议 | 只显示1~2行 |
| 系统消息 | 3秒后淡出 |
| 错误提示 | 红色短文本，不弹大窗 |

---

## 5.4 自动收起规则

左侧信息面板默认半透明。  
如果战斗激烈或AI靠近左侧区域：

```text
左侧面板透明度降低到30%
鼠标靠近或点击后恢复100%
```

移动端可直接折叠为小按钮。

# 6. 右侧AI状态区 RightInfoPanel

## 6.1 右侧信息用途

右侧用于显示AI当前状态，不承担主要操作。

显示内容：

1. AI当前流派
2. 当前正在释放的攻击
3. 最近获得的BD
4. AI身上的Buff / Debuff
5. 即将触发的大招提示
6. 当前克制关系摘要

---

## 6.2 布局结构

```text
┌──────────────┐
│ AI状态         │
│ 流派：雷电连奏 │
│ 攻击：星火连射 │
├──────────────┤
│ Buff/Debuff    │
│ [护盾] [加速]  │
│ [禁疗] [减速]  │
├──────────────┤
│ 危险提示       │
│ 大招充能 80%   │
└──────────────┘
```

---

## 6.3 右侧面板折叠规则

右侧面板必须可折叠。

```text
默认显示：简洁模式
点击展开：详细模式
战斗激烈：自动半透明
```

### 简洁模式

只显示：

- AI流派
- 当前攻击名
- 2~4个关键Buff
- 危险等级

### 详细模式

显示：

- 全部Buff
- 最近BD
- 当前抗性
- 当前弱点
- 当前AI行为倾向

# 7. AI音效节奏区 AIRhythmPanel

## 7.1 系统定位

这是项目核心特色UI。  
用于显示AI当前攻击的音效节奏、弹幕节拍、预警和后摇。

必须放在底部战斗区下方，不能遮挡主战斗。

---

## 7.2 布局结构

```text
┌────────────────────────────────────────────┐
│ AI攻击：星火连射        危险：中             │
│ 高音轨  ━●━●━●━●━●━                     │
│ 中音轨  ━━━●━━━━●━━━                     │
│ 低音轨  ━━━━━━━●━━━━                     │
│ 进度    ███████░░░░░  1.8s / 3.0s          │
└────────────────────────────────────────────┘
```

---

## 7.3 组成元素

| 元素 | 说明 |
|---|---|
| 当前攻击名 | 显示AI正在释放什么攻击 |
| 危险等级 | 低 / 中 / 高 / 极高 |
| 高音轨 | 高频弹幕、快速弹 |
| 中音轨 | 普通弹幕、常规技能 |
| 低音轨 | 重击、AOE、大招 |
| 进度条 | 当前攻击时间轴 |
| 播放指针 | 当前节拍位置 |
| 后摇窗口 | 攻击结束后的可反制时间 |
| 节拍点 | 每个弹幕/音效触发点 |

---

## 7.4 尺寸规则

| 平台 | 高度 |
|---|---:|
| PC | 70 ~ 90px |
| 手机横屏 | 80 ~ 110px |
| 平板 | 90 ~ 120px |

---

## 7.5 颜色规则

| 音轨 | 颜色 |
|---|---|
| 高音轨 | 蓝 / 洋红 |
| 中音轨 | 紫 / 粉 |
| 低音轨 | 橙 / 红 |
| 危险节拍点 | 红色闪光 |
| 后摇窗口 | 绿色或浅黄色 |
| 当前播放指针 | 白色亮线 |

---

## 7.6 后摇提示

后摇提示要非常清楚，因为玩家要在这里释放技能或召唤精英。

显示方式：

```text
进度条末端出现一段浅黄色区域
鼠标悬停显示：AI后摇，可反击
```

示例：

```text
███████████▓▓
            ↑
        后摇窗口
```

---

## 8.2 布局结构

┌────────────────────────────────────────────┐
│ 当前BD  8 / 20        [全部▼]                │

---

## 8.3 显示规则

| 项目 | 规则 |
|---|---|
| Icon大小 | PC 42×42，移动端48×48 |
| 默认显示数量 | 8~10个 |
| 超出显示 | 横向滚动 |
| 核心BD | 固定在最左侧 |

```

| 平台 | 宽度 |
|---|---:|

# 11. 造物主技能区 CreatorSkillPanel

## 11.1 技能区定位

造物主技能是玩家主动干预战斗的核心操作。

需要放在右下方，方便点击，但不能太靠近屏幕边缘。

# 27. UI动效规则

透明度从0到1
```

不要超过0.3秒，避免影响操作。

# 28. UI信息密度控制

## 28.1 一屏最多常驻信息

常驻信息不要超过以下数量：

| 类型 | 最大数量 |
|---|---:|
| 顶部状态项 | 6个 |
| 左侧提示 | 2条 |
| 右侧状态 | 4行 |
| BD图标 | 10个可见 |
| 怪物按钮 | 8个可见 |
| 技能按钮 | 8个 |
| 浮动提示 | 3条 |
| 冒泡 | 2个 |

---

## 28.2 信息过载处理

如果信息过多：

1. 合并小伤害跳字。
2. 折叠左侧提示。
3. 右侧只显示危险状态。
4. BD列表横向滚动。
5. 怪物抽屉分页。
6. 普通Buff隐藏，只显示关键Debuff。
7. UI透明度降低，让战斗优先。

# 30. 最终UI体验目标

玩家进入战斗后，应该能自然形成以下阅读顺序：

```text
看顶部 → 知道AI血量、时间、能量
看中央 → 判断战场局势
看节奏条 → 判断AI正在放什么攻击
看BD列表 → 判断AI当前流派
看底部 → 选择怪物或技能
看预警 → 选择释放时机
看结算 → 明白自己为什么赢/输
```

最终界面要达到：

1. 中央战斗清楚。
2. 底部操作顺手。
3. AI信息不藏起来。
4. 玩家能判断克制关系。
5. 造物主技能释放有明确反馈。
6. UI有音乐控制台风格。
7. 不因为信息多而混乱。
8. 移动端也能舒服操作。
9. PC端能用快捷键提高效率。
10. 整体像一场失控演唱会的后台控制界面。
